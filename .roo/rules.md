# EcoLogic Manager — Roo Rules

## Project Context
Read `COPILOT_INSTRUCTIONS.md` in the workspace root for full documentation.

## Architecture Summary
Three-tier IoT system:
1. **Firmware** — ESP8266 Arduino C++ in `EcoLogic_manager/`
2. **Shared Frontend** — `HTML_data/` (serves BOTH the device and the FastAPI server)
3. **Backend API** — Python FastAPI in `standalone_server/` (MySQL + InfluxDB + Grafana)

## File Editing Rules

### NEVER edit these (generated/output):
- `EcoLogic_manager/data/` — gzip output, regenerated by `make_gz.sh`
- `EcoLogic_manager/build/` — compiled firmware binary
- `standalone_server/.env` — secrets, gitignored

### Source of truth for HTML/JS/CSS:
- Always edit `HTML_data/` — not `EcoLogic_manager/data/`
- After editing `HTML_data/`, run `make_gz.sh` to update device filesystem

## Coding Patterns

### Dual-mode frontend detection
Every JS file uses this pattern — preserve it in all changes:
```javascript
const IS_SERVER = window.location.pathname.startsWith('/api/');
const API_PREFIX = IS_SERVER ? '/api' : '';
// All fetch() calls must use API_PREFIX
```

### FastAPI endpoints
- All routes must use `/api/` prefix (Nginx reverse proxy requirement)
- Arduino short params: `id` = device_id, `tk` = token, `ip` = IP address
- DB connection via `get_conn()` — always wrap in try/finally and close connection

### Arduino C++ firmware
- Global pin arrays: `pin[]`, `pinmode[]`, `stat[]`, `defaultVal[]`, `descr[]` — size = N_WIDGETS
- Feature flags are `#define` in `EcoLogic_manager.ino` — compile-time only
- Device ID = MAC address without colons (set by `generate_device_id()`)
- Filesystem is LittleFS (not SPIFFS) by default
